Functions

WHAT ARE FUNCTIONS?

Functions are the actions and behaviors of code; functions are sometimes also known as subroutines. Functions can be given names so they can be reused (or they can be anonymous), and they usually perform a specific task. Some functions return values while others perform an operation.

Advantages for using functions in your application:

  • Functions help break a complex programming task into simpler steps.
  • DRY - Don’t repeat yourself: functions help in reducing duplicate code and allow for reuse across multiple programs.
  • Functions make it easier to find bugs or problems within code.

Defining a function

There are several ways to write a function declaration. Most broadly, they include a name, a list of arguments, and a code block that runs when the function is called. There are several ways to define functions in avaScript, detailed in the function guide.

  1. // Function declaration - JavaScript
  2. const add = (num1, num2) => {
  3. return num1 + num2;
  4. }

Using a function

To use (call or invoke) a function, you reference it by its name with a set of parentheses (). If the function is defined with arguments, the parentheses will include the additional necessary information.

  1. // Call a function
  2. let sum = add(2, 3); // perform the add function on 2 and 3 and store
  3. // the result in a variable called sum

Remember, if you want to examine the output of a function when you’re debugging, you’ll want to print the output of the function call to the console. The output of this function was stored in a variable called sum, so print the contents of the variable sum to see whether the function is executing correctly.

  1. console.log(sum) // prints the value stored in our variable to the console.

WHEN DO WE USE FUNCTIONS?

Functions are used in our web applications that involve any kind of interactivity or interaction. As programs grow in complexity, having our code broken up into functions helps to keep our application organized and reduces the overall complexity of our program.

If the operation you are writing appears more than once in your application, you should turn it into a function and reuse that one function. If your code has to be manipulated or an action needs to be made, a function should be created.

Again, a function should perform only one task. When a function becomes too long or too hard to understand, it should be split up into multiple functions.

Linking a JavaScript File

In order for a JavaScript file (.js) to be used by an HTML document, the two files have to be linked with a <script> element. Just like with CSS, without the link, the HTML file would have no knowledge of the JavaScript file, and the two would not be connected. However, unlike CSS which is linked in the <head> of the HTML document, JavaScript files are usually linked toward the bottom of an HTML page. This is so the page can first load the content before spending time loading external .js files. You may also see <script> tags in the head of a document; these are often functions which are involved in rendering an HTML page.

To link an HTML file to a JavaScript file, we need to add a <script> element with one attribute:

  1. src is the filename (and location) of the .js file
  • <script src="./script.js"></script>
